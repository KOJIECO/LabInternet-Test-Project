## 1. Инструкция по установке и запуску

### Шаги по установке и запуску

1. **Клонирование проекта и переход в рабочую директорию**:
   ```bash
   cd LabInternet-Test-Project
   ```

2. **Создание и активация виртуального окружения**:
   - **Windows**:
     ```bash
     python -m venv venv
     venv\Scripts\activate
     ```
   - **Linux/macOS**:
     ```bash
     python3 -m venv venv
     source venv/bin/activate
     ```

3. **Команды для установки зависимостей**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройка переменных окружения**:
   Создайте файл `.env` в корневой папке проекта на основе шаблона `.env.template`:
   ```env
   PORT=8000
   HOST=0.0.0.0
   ENVIRONMENT=development
   DATABASE_URL=sqlite+aiosqlite:///./sql_app.db
   GEMINI_API_KEY=ваш_ключ_от_gemini_api
   RATE_LIMIT_REQUESTS=5
   RATE_LIMIT_WINDOW=60
   USE_MOCK_EMAIL=True
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=
   SMTP_PASSWORD=
   SMTP_FROM_EMAIL=no-reply@portfolio.dev
   CONTACT_RECEIVER_EMAIL=owner@portfolio.dev
   ```

5. **Сидирование демонстрационных данных** (создаст таблицы БД и заполнит 5 тестовыми записями):
   ```bash
   python seed_data.py
   ```

6. **Запуск локального сервера**:
   ```bash
   python run.py
   ```
   После этого веб-интерфейс дашборда будет доступен по адресу: `http://localhost:8000`

7. **Запуск тестов**:
   ```bash
   pytest
   ```

### Паттерны проектирования
1. **Repository Pattern (Репозиторий)**: Инкапсулирует логику запросов к БД в `ContactRepository`. Часть с бизнес-логикой не видит деталей реализации СУБД.
2. **Service Layer (Слой сервисов)**: Выделяет бизнес-процессы в единый пайплайн в `ContactService`.
3. **Dependency Injection (Внедрение зависимостей)**: Сессии базы данных инжектируются в хендлеры FastAPI через встроенный механизм `Depends(get_db)`.
4. **Sliding Window (Скользящее окно)**: Алгоритм ограничения частоты запросов реализован в оперативной памяти на базе очередей временных меток клиентов.

### Объяснение выбора технологий
- **FastAPI**: Быстрая обработка, встроенная документация и валидация данных.
- **SQLAlchemy 2.0**: ORM с поддержкой асинхронности.
- **SQLite / PostgreSQL**: Быстрый локальный запуск, PostgreSQL легко развертывается.

## 4. Реализация API

#### 1. `POST /api/contact` — Отправка контактной формы
Принимает форму, валидирует поля, отправляет на анализ ИИ, сохраняет обогащенную запись в БД и инициирует отправку уведомлений. Защищен лимитом запросов.

- **Пример запроса**:
  ```json
  {
    "name": "Алексей HR",
    "email": "alex.hr@company.ru",
    "phone": "+79991112233",
    "message": "Здравствуйте! Заинтересовал ваш профиль. Хотим предложить вакансию FastAPI разработчика."
  }
  ```
- **Пример ответа**:
  ```json
  {
    "id": 6,
    "name": "Алексей HR",
    "email": "alex.hr@company.ru",
    "phone": "+79991112233",
    "message": "Здравствуйте! Заинтересовал ваш профиль. Хотим предложить вакансию FastAPI разработчика.",
    "sentiment": "positive",
    "category": "job_offer",
    "ai_response": "Здравствуйте, Алексей! Спасибо за предложение. Я заинтересован в обсуждении вакансии.",
    "created_at": "2026-06-20T13:00:00Z"
  }
  ```

#### 2. `GET /api/metrics` — Статистика системы
Возвращает агрегированные данные для графиков панели управления и список 5 последних сообщений.

- **Пример ответа**:
  ```json
  {
    "total_messages": 1,
    "sentiment_distribution": {
      "positive": 1,
      "neutral": 0,
      "negative": 0
    },
    "category_distribution": {
      "job_offer": 1,
      "question": 0,
      "partnership": 0,
      "spam": 0,
      "other": 0
    },
    "recent_messages": [
      {
        "id": 6,
        "name": "Алексей HR",
        "email": "alex.hr@company.ru",
        "phone": "+79991112233",
        "message": "Здравствуйте! Заинтересовал ваш профиль...",
        "sentiment": "positive",
        "category": "job_offer",
        "ai_response": "Здравствуйте, Алексей!...",
        "created_at": "2026-06-20T13:00:00Z"
      }
    ]
  }
  ```

#### 3. `GET /api/health` — Мониторинг состояния
Используется для проверки работоспособности сервиса, подключения к БД и состояния API-ключа Gemini.

- **Пример ответа**:
  ```json
  {
    "status": "healthy",
    "database": "healthy",
    "ai_service": "active"
  }
  ```

### Валидация и обработка ошибок
- Валидация номера телефона выполняется с помощью регулярного выражения Pydantic `@field_validator("phone")`, где можно использовать от 7 до 15 цифр, опциональный знак `+` и разделители.
- Имя проверяется на наличие непустых символов. Email проверяется на соответствие стандарту RFC 5322.
- Обработчик исключений перехватывает системные ошибки, логирует трассировочный стек в файл и возвращает пользователю безопасный JSON с кодом `500 Internal Server Error`:
  ```json
  {"detail": "Внутренняя ошибка сервера. Пожалуйста, попробуйте позже."}
  ```

## 5. AI-интеграция

### Задачи ИИ в проекте
Модель ИИ используется для автоматического разбора входящих писем, снижения нагрузки на разработчика и мгновенной демонстрации интерактивного взаимодействия работодателю.

### Промпт для Google Gemini
```text
Вы — интеллектуальный AI-ассистент разработчика. Проанализируйте следующее сообщение от пользователя по имени "{name}":
---
{message}
---

Верните строго JSON-объект со следующими полями без какого-либо дополнительного текста, markdown-разметки или backticks (```json):
{
    "sentiment": "positive" | "neutral" | "negative",
    "category": "job_offer" | "question" | "partnership" | "spam" | "other",
    "suggested_reply": "Ваш вежливый, персонализированный ответ пользователю на русском языке"
}

Особенно важно:
- В поле "sentiment" должно быть ровно одно из слов: positive, neutral, negative.
- В поле "category" должно быть ровно одно из слов: job_offer, question, partnership, spam, other.
- В поле "suggested_reply" сгенерируйте вежливый ответ от лица разработчика (Eduard)...
```

### Реализация Fallback-режима (Отказоустойчивость)
Если лимиты API-ключей исчерпаны или отсутствует интернет-соединение, активируется локальный парсер:
- **Определение тональности**: подсчет вхождения ключевых эмоциональных по тегам.
- **Категоризация**: поиск по регулярным выражениям и ключевым словам (работа, hr, оффер -> `job_offer`; проект, сотрудничество -> `partnership` и т.д.).
- **Генерация ответа**: выбор подходящего шаблона ответа из словаря на основе определенной категории.

## 6. Что сделано с помощью AI

### Какие части кода генерировались
- Логика построения локального эвристического классификатора в методе `_get_fallback_analysis` в `app/services/ai.py`.
- Базовая CSS-сетка для дашборда аналитики и визуальное оформление AI-терминала на фронтенде.

### Использованные промпты
- *"Напиши локальную fallback функцию на Python, которая по ключевым словам классифицирует текст сообщения на категории (job_offer, question, spam) и определяет тональность без внешних библиотек."*
- *"Создай стили для блока с отображением AI-логирования, стилизованный под темный терминал Linux с подсветкой вывода."*

### Что пришлось исправлять вручную
- **Жизненный цикл сессий SQLAlchemy**: ИИ генерировал синхронные вызовы коммитов внутри асинхронных методов сервиса, пришлось переписывать под контексты сессий.
- **Интеграция лимитера**: Генерация декоратора требовала исправления передачи параметров запроса `Request` в FastAPI Dependency Injection.
- **Спецификация Pydantic**: Вручную обновил конфигурацию моделей с устаревшего стиля Pydantic v1 `class Config:` на современный v2 `model_config = ConfigDict(from_attributes=True)`.
- **Некорректная обработка JSON**: ИИ неверно обрабатывал JSON в Python, внося лишние запятые.

## 7. Хранение данных и безопасность

### Логирование
Хранение логов организовано через модуль `logging` стандартной библиотеки Python с использованием `RotatingFileHandler`:
- Логи записываются в директорию `logs/app.log`.
- Установлен максимальный размер файла в 10 МБ с ротацией в 5 архивных копий, чтобы избежать переполнения диска на сервере.
- Формат записи логов содержит метку времени, уровень критичности, имя модуля и сообщение.

### Rate Limiting (Ограничение частоты)
- Реализован в `app/core/security.py` с использованием алгоритма скользящего окна.
- Ограничение по умолчанию: 5 запросов в течение 60 секунд с одного IP-адреса.
- Превышение лимита вызывает исключение `429 Too Many Requests`.

### Статистика обращений
- Вся статистика и записи сообщений хранятся в реляционной базе данных.
- Для панели управления метрики не дублируются в памяти, а вычисляются динамически при обращении к `/api/metrics` с помощью SQL-агрегаций, что гарантирует актуальность и целостность данных в многопользовательской среде.
